# はじめに{#getting-started}

この情報は、Adobe Mobile Services の理解と使用に役立ちます。

AdobeMobile Servicesは、次の要素で構成されています。

* Adobe Mobile Services UI
* Adobe Mobile SDK

Adobeは、ユーザーの関与を増やし、モバイルアプリへの投資に対する投資利益率(ROI)を証明する最も効果的な方法を探し求めている企業会社に対して、アプリユーザーを獲得、関与させ、エクスペリエンスを分析、最適化するエンドツーエンドソリューションを提供します。

現在、モバイルアプリの横置きは、スマートフォンが初めて起動されたときとは大きく異なっています。 モバイルアプリを使用して顧客をブランドに結び付けるだけでは不十分です。今日では、様々なチャネルにわたって、まとまった魅力的な顧客エクスペリエンスを推進し、最も忠実で価値の高い顧客を惹きつける戦略的なタッチポイントとしてモバイルアプリを使用する必要があります。 しかし、これらのユーザーがアプリを操作できるようにするには、説得力のあるコンテンツ、コンテキスト通知、インテリジェントなパーソナライゼーション、統合されたアプリ分析などが必要です。

## Adobe Mobile Services UI {#mobile-services-ui}

Mobile Services UIは次のブラウザーでサポートされています。

* Google Chrome（最新の2つのバージョン）
* Mozilla Firefox（最新の2つのバージョン）
* Apple Safari（最新の2つのバージョン）
* Microsoft Edge（最新の2つのバージョン）

Adobe Mobile Services は、次の方法でモバイルアプリのエンゲージメントの促進を支援します。

### 獲得

*獲得*&#x200B;では、有料メディア、オウンドメディアおよび無料メディアを使用して、ユーザーの獲得を推進し、主要なアプリストアからのアプリダウンロードを促します。AdobeのMobile Servicesを使用すると、アプリのユーザー獲得プロセスを高速化できます。

AdobeMobile Servicesは、獲得追跡やディープリンクなどのユーザー獲得ワークフローを提供し、アプリユーザー獲得におけるチャネルの効果を測定します。 どのユーザーがどのチャネルから来たかを追跡するマーケティングリンクを使用すると、利益につながり、積極的に関与するユーザーを集めるうえで、どの獲得チャネルが最も効果的かを明確に把握することができます。

さらに、ディープリンクを使用すると、ユーザーを表示したいアプリコンテンツに直接誘導し、必要に応じてユーザーにアプリのインストールを促すことができます。

獲得オファーには、次の主な機能があります。

* アプリの獲得分析
* アプリストア間のリンクの追跡
* アプリへのディープリンク
* 広告ネットワークとの再統合

このフェーズについて詳しくは、 [獲得を参照してください](/help/using/acquisition-main/acquisition-main.md)。

### 分析

*分析*&#x200B;では、消費者のモバイルアプリの使い方およびコンバージョンや復帰の要因を把握できます。

Adobe Analyticsを使用すると、ユーザーがアプリをダウンロード、インストール、開く方法のファネルに関する主なインサイトを得ることができます。 また、アプリのコンテンツとUIを測定および分析し、コホート分析、パスおよびフォールアウトを実行できます。 Adobe Analyticsでは、中央のデータストアを使用して、マーケティングに関する意思決定を知らせ、組織内のマーケティングデータのサイロを減らすことができます。

Adobe Audience Managerを使用すると、オーディエンスセグメントをリッチデータと共に拡張し、よりコンテキスト的で個人的なエクスペリエンスを提供できます。

*分析*&#x200B;では、次の主要な機能を提供します。

* アプリエンゲージメントの分析
* パスとファネルの分析
* コホートとリテンションの分析
* 場所の分析
* 広範なデバイスとプラットフォームのサポート

顧客を実行および分析できるレポートの詳細については、「 [レポート](/help/using/usage/usage.md)」を参照してください。

### エンゲージメント

*エンゲージメント*&#x200B;では、関連するプッシュ通知およびアプリ内メッセージを使用して、ユーザーとコミュニケーションを取ることができます。ターゲット設定されたプッシュ通知とアプリ内メッセージを使用すると、ユーザーが引き続きアプリに戻るようにできます。 Analyticsのセグメントがサポートされ、プッシュ通知を応答するユーザーセグメントにターゲットして、コンバージョン率を向上させることができます。

*エンゲージメント*&#x200B;では、次の主要な機能を提供します。

* プッシュ通知は、Analyticsセグメントによってトリガーされます。
* アプリ内メッセージは、リアルタイム分析、アラートおよび新しいオファー/コンテンツによってトリガーされます。
* 表示、クリックスルー率およびダウンストリーム行動を把握

### Adobe Mobile メッセージ

プッシュおよびアプリ内通知を使用して、ユーザーと通信できます。 プッシュ通知は、デバイスのオペレーティングシステムを介して送信されます。一方、アプリ内メッセージは、ユーザーがアプリとの対話操作を行うとアプリ内で送信されます。 通常、アプリ内メッセージには、ポップアップやインタースティシャルなど、様々な追加形式が含まれます。

Adobeモバイルでは、次のタイプのメッセージを設定できます。

**プッシュ通知**（アプリの外部に表示される）には、次の機能がオファーされます。

* 関連するプッシュ通知による再関与の促進
* ブランドのアプリをダウンロードし、オプトインを通じて受け入れられた顧客に対して、プッシュ通知の受信に関するメッセージを作成、セグメント化および送信します。
* モバイルアプリからではなく、アプリストアによってサーバー側に送信されます。

For more information about creating push notifications, see [Create a push message](/help/using/in-app-messaging/t-create-push-message/t-create-push-message.md).

**アプリ内通知** オファーには次の機能があります。

* ユーザーがアプリセッション中に、特定のアクションに移動する。
* メッセージは、プッシュ配信ネットワークではなく、アプリによって配信されるので、追加の形式（警告、フルスクリーン）が必要です。
* リアルタイム分析によってトリガーされます。
* アプリと製品のクロスプロモーションを許可します。
* ユーザーにアプリストアのレーティングを終了するよう勧めます。
* リアルタイムの場所ベースのメッセージを配信

アプリ内メッセージの作成について詳しくは、アプリ内メッセージの [作成を参照してください](/help/using/in-app-messaging/t-in-app-message/t-in-app-message.md)。

### 最適化

**&#x200B;では、コンバージョン（購読、コマース、広告収入など）を最適化して、顧客維持を強化します。アプリでのユーザーエクスペリエンスを最適化すると、コンテンツをパーソナライズしてROIとコンバージョンを最大限に高めることができます。

テストおよび Adobe Target について詳しくは、[Adobe Target](https://docs.adobe.com/content/help/ja-JP/target/using/target-home.html) に移動してください。

### ジオターゲティング

モバイルデバイスを使用すると、ユーザーがアプリに関与している場合や、バックグラウンドでGPS位置を通じて動作している場合に、本質的にユーザーがどこにいるかを把握できます。 GeoTargetingを使用すると、Adobe Targetは、近接性が重要な場合に、カスタマイズされた関連コンテンツ、オファー、メッセージを同時に配信できます。 目標地点の定義した範囲内にいるユーザーや、iBeaconに近い場合、関連するプッシュ通知を使用してターゲットできます。

Adobe Target for Mobileアプリでは、Adobe Analyticsで利用できる強化されたセグメント化とレポートを最大限に活用できるようになりました。 つまり、Adobe Targetは、Analyticsの主要なアプリ指標を使用してターゲットやパーソナライズを行うことで、それらの指標をすべて活用できます。また、テストの成功に関するより深いレポートを得るために、マーケターは「whit」という質問（アプリへの投資利益率を示すプレッシャーを感じるアプリマーケティング担当者を回避する可能性のある回答）をより深く理解できます。 アプリ統合のAnalytics/ターゲットは、組み合わせオファーを提供します。これは、市場で利用可能な最も堅牢なアプリエンゲージメントソリューションです。

ロケーションについて詳しくは、次のコンテンツを参照してください。

* [Mobile Services ユーザーガイド内の場所](/help/using/location/c-location-overview.md)
* [Android SDK ガイドの ロケーション](/help/android/location/location.md)
* [iOS SDK ガイドの ロケーション](/help/ios/location/location.md)

## Adobe Mobile SDK {#mobile-services-sdk}

アドビでは、これらすべての領域にわたる顧客エンゲージメントを加速するエンドツーエンドのモバイルマーケティングソリューションを提供しています。1つのSDKで、Adobe Analytics、Adobe Campaign、Adobe Audience Managerの機能にアクセスできるので、複数の異なるSDKの管理に伴う技術的なコストを削減できます。

AdobeモバイルSDKオファーには、次の機能があります。

* エンドツーエンドのモバイルエンゲージメント

   軽量で統合された1つのAdobeMobile SDKを使用して、様々なプラットフォームにわたるアプリを測定および最適化できます。
* 新しい顧客を獲得し、魅力的なマイクロな瞬間を提供

   * リッチメディアのサポートやアプリ内メッセージなど、ターゲットを設定したプッシュ通知を通じて、ユーザーを何度もアプリに戻すことができます。
   * ディープリンクを使用すると、アプリのユーザーを目的のコンテンツに直接誘導できます。

* ROIを引き上げるためのエクスペリエンスの測定と最適化

   ファネル分析（ダウンロード、インストール、開く）、セッション長、クラッシュ、ビーコン、メッセージングインタラクションなどのアクションを含む、アプリのライフサイクル指標を把握できます。
* 包括的

   * 主要なモバイルオペレーティングシステムとクロスプラットフォーム開発ツールに対する幅広いサポート。
   * スマートフォン、タブレット、ウェアラブル、OTT（オーバーザトップ）コンソールにまたがる広範なデバイスサポート。

* 統合

   * 複数のソリューション(Analytics、キャンペーンおよびAudience Manager)に対する1つのSDKで、開発者の導入時間と労力を削減できます。
   * 「ベースライン」アプリのライフサイクル指標を収集するために必要なコードは1行のみです。
   * モバイル戦略の発展に伴い、Adobe Experience Cloudの機能を簡単にアクティブ化して、ユーザーを獲得、分析、エンゲージメントを図ることができます。

* 高速で軽量

   * Adobeサーバーやサードパーティ製システムにデータを送信する際の、デバイスの処理負荷を最小限に抑えます。
   * 小さなフットプリントは、アプリストアに送信されるアプリパッケージのサイズを最小限に抑えます。

For more information about the Adobe Mobile SDKs, see [Android SDK 4.x for Experience Solutions](https://docs.adobe.com/content/help/ja-JP/mobile-services/android/overview.html) and [iOS SDK 4.x for Experience Cloud Solutions](https://docs.adobe.com/content/help/ja-JP/mobile-services/ios/rel-notes.html).
